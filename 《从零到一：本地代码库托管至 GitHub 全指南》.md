# 《从零到一：本地代码库托管至 GitHub 全指南》

## 第一阶段：GitHub 远程端配置（创建仓库）

这是为了在云端开辟一个存放代码的“保险柜”。

1. **登录 GitHub**：访问 [github.com](https://github.com/) 并登录。
2. **新建仓库**：
   - 点击右上角 **"+"** 号，选择 **New repository**。
   - **Repository name**：填写你的项目名（如 `Go-Learning`）。
   - **Public/Private**：根据需要选择公开或私有。
   - **⚠️ 关键点**：**不要**勾选 `Add a README file`、`Add .gitignore` 或 `Choose a license`。我们要保持仓库为“纯空壳”，以避免后续的同步冲突。
   - 点击 **Create repository**。
3. **获取地址**：在创建成功的页面，复制那个以 `.git` 结尾的链接（例如 `https://github.com/用户名/Go-Learning.git`）。

## 第二阶段：本地仓库初始化与绑定

在你的电脑上，将“代码文件夹”变成一个“受 Git 管理的仓库”。

1. **打开终端**：进入你的代码文件夹 `C:\Users\Aurora\Desktop\Go Project`，在地址栏输入 `cmd` 并回车。

2. **初始化**：

   Bash

   ```
   git init
   ```

   *(此时文件夹内会生成一个隐藏的 `.git` 目录)*

3. **配置忽略规则（防止垃圾文件上传）**：

   在文件夹内新建一个文本文件，重命名为 `.gitignore`（注意前面有个点，不要带 .txt 后缀），填入以下内容并保存：

   Plaintext

   ```
   *.exe
   *.exe~
   *.dll
   *.so
   *.dylib
   .idea/
   .vscode/
   *.log
   ```

## 第三阶段：建立连接与首次推送

将本地“档案室”与云端“保险柜”连接起来。

1. **添加文件到暂存区**：

   Bash

   ```
   git add .
   ```

2. **提交快照到本地历史库**：

   Bash

   ```
   git commit -m "Init: 首次上传我的 Go 语言项目"
   ```

3. **重命名分支**（确保与 GitHub 默认分支名称匹配）：

   Bash

   ```
   git branch -M main
   ```

4. **绑定远程仓库**：

   Bash

   ```
   git remote add origin <粘贴你在第一阶段复制的链接>
   ```

5. **推送到远程**：

   Bash

   ```
   git push -u origin main -f
   ```

   *(注：第一次使用 `-f` 是为了强制覆盖云端那个默认的空状态，以后不需要 `-f`)*

## 第四阶段：日常维护（三部曲）

当你以后修改了代码（比如修改了 `57.http服务端.go`），只需要执行这三行：

Bash

```
# 1. 收集所有改动
git add .

# 2. 存入本地历史记录（备注要清晰）
git commit -m "Update: 修改了服务端路由逻辑"

# 3. 同步到 GitHub 云端
git push
```

## 常见问题处理速查

| **问题**                    | **解决方案**                                                 |
| --------------------------- | ------------------------------------------------------------ |
| **提示权限不足**            | 确保你已在 GitHub 网页版授权了你的电脑，如果是第一次，会弹出网页登录窗口，正常点击授权即可。 |
| **提示拒绝推送 (rejected)** | 通常是因为云端有文件（README）你本地没有。请确认你是否已执行 `git push -u origin main -f`（强制推送）。 |
| **如何查看改了什么**        | 随时执行 `git status`，它会用红色列出已修改的文件，用绿色列出已暂存的文件。 |
| **Commit 备注怎么写专业**   | 推荐前缀：`Feat` (新功能), `Fix` (修 Bug), `Update` (常规更新), `Docs` (修说明文档)。 |
