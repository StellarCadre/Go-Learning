当前项目的启动说明：
服务端启动步骤：
打开命令行终端（CMD/PowerShell），进入项目源码所在文件夹。
执行启动命令，一次性编译运行所有依赖文件：go run main.go Server.go user.go。启动成功标志：终端无报错、无主动退出，服务处于阻塞监听状态，等待客户端接入。
服务默认配置：监听 127.0.0.1:8000，仅本机设备可连接；如需局域网其他设备访问，将监听地址改为 0.0.0.0:8000。
客户端连接测试（两种方式任选）：
方式 1：系统 telnet 工具（快速测试）：
新开独立 CMD 窗口，解决中文乱码优先执行编码切换命令：chcp 65001
发起 TCP 连接：telnet 127.0.0.1 8000
连接成功表现：客户端窗口自动弹出对应用户上线广播消息，多客户端连接时，新用户上线会对所有在线客户端推送上线提醒

版本4：用户业务层封装
将Server.go中的用户上线、下线、把用户处理业务的逻辑，重新封装到user.go中
版本对比：版本 3（消息广播版）→ 版本 4（用户业务封装版）完整差异清单
即：
Online()：用户上线全套逻辑
旧版：写在 handleConn 里，手动加锁、写入 OnlineMap、调用 Broadcast
新版：全部封装进 User.Online，外部只需要一行 user.Online()
s.mapLock.Lock()
s.OnlineMap[user.Name]=user    ==> user.Online()
s.mapLock.Unlock()

Offline()：用户下线全套逻辑
旧版：读协程里手写加锁 delete、Broadcast 下线消息
新版：封装到 User.Offline，外部一行调用即可
s.Broadcast(user,"This user is offline")    ==>   user.Offline()

DoMessage(msg string)：客户端消息处理
旧版：读协程直接调用s.Broadcast(user,message)
新版：统一交给 User.DoMessage，未来加过滤、指令、私聊都只改这里
s.Broadcast(user,message)    ==>   user.DoMessage(message)
并且，User 结构体新增字段server *Server，让每个 User 持有所属服务端实例，不用每次传 server 参数。相应的NewUser 构造函数参数变更。

双方沟通全展示：
根据通信系统架构图，可知分为server和client两部分，server负责处理用户的请求，client负责展示用户的界面。
对于server部分，有一个OnlineMap，存储在线用户的信息，包括用户的id、用户本身。此外，还有一个Message部分，是一个channel，用来向所有用户广播消息，也能接收用户信息。
对于client部分，有多个client，每个都是一个用户，会使用用户类实例化每一个具体用户。并且，每个用户还有自己的user对象，用在自己的channel上，表示该用户使用自己的channel接收服务器的消息。
